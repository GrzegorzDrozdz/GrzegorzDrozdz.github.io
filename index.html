💳 Credit Card Customer Segmentation 💳 Segmentacja Klientów Kart Kredytowych

This project utilizes Unsupervised Machine Learning techniques to analyze the financial behaviors of ~9,000 active credit card holders. By applying the K-Means algorithm (k=5), the model segments customers into distinct profiles to facilitate targeted marketing strategies and risk assessment.

Ten projekt wykorzystuje techniki Uczenia Nienadzorowanego do analizy zachowań finansowych około 9000 aktywnych posiadaczy kart kredytowych. Przy użyciu algorytmu K-Means (k=5), podzieliłem bazę na odrębne profile ułatwiające optymalizację kampanii marketingowych oraz ocenę ryzyka.

Methodology & Workflow Metodologia i Etapy Pracy
1. Exploratory Data Analysis (EDA) 1. Eksploracyjna Analiza Danych (EDA)

Understanding the raw data structure. Zrozumienie struktury surowych danych.

  • Analyzed distributions using histograms and boxplots.
  • Identified heavy right-skewness and outliers typical for financial data.
  • Investigated correlations between specific financial features (e.g., Purchases vs. One-Off Purchases).
  • Analiza rozkładów przy użyciu histogramów i wykresów pudełkowych.
  • Zidentyfikowano silną prawoskośność i wartości odstające typowe dla danych finansowych.
  • Zbadano korelację między poszczególnymi atrybutami (np. zakupy całkowite a jednorazowe).
2. Advanced Preprocessing 2. Zaawansowane Przetwarzanie

Preparing data for distance-based algorithms. Przygotowanie dla algorytmów opartych na odległości.

  • Imputation: Handled missing values in MINIMUM_PAYMENTS and CREDIT_LIMIT.
  • Yeo-Johnson Transform: Crucial Step Applied Power Transformer to stabilize variance. This significantly improved clustering performance over standard scaling.
  • Scaling: Final normalization using StandardScaler.
  • Imputacja: Uzupełniono braki danych w kolumnach minimalnych płatności oraz limitów.
  • Transformacja Yeo-Johnsona: Kluczowy Krok Zastosowano transformację potęgową do stabilizacji wariancji. Poprawiło to jakość klastrowania znacznie skuteczniej niż samo skalowanie.
  • Skalowanie: Ostateczna normalizacja przy użyciu StandardScaler.
3. Dimensionality Reduction (PCA) 3. Redukcja Wymiarowości (PCA)

Reducing noise and complexity. Zmniejszenie złożoności zbioru danych.

  • Utilized Principal Component Analysis (PCA).
  • Effectively reduced noise and multicollinearity.
  • Compressed the feature space while retaining the vast majority of data variance.
  • Wykorzystano technikę PCA (Analiza Głównych Składowych).
  • Skutecznie zredukowano szum i współliniowość zmiennych.
  • Skompresowano przestrzeń cech, zachowując przy tym zdecydowaną większość wariancji danych.
4. Algorithm Selection 4. Wybór Algorytmu

Why K-Means over others? Dlaczego wybrano K-Means?

  • Hierarchical: Used dendrograms to estimate the natural number of clusters.
  • DBSCAN (Rejected): Classified high-spending clients as "noise", which is undesirable for VIP business profiles.
  • K-Means (Selected): The optimal number of clusters (k=5) was determined using the Elbow Method and Silhouette Score.
  • Dendrogram: Użyte pomocniczo przy estymacji naturalnej liczby skupień.
  • DBSCAN (Odrzucony): Algorytm potraktował klientów o najwyższych wydatkach jako "szum", co dla profili VIP jest błędem biznesowym.
  • K-Means (Wybrany): Ostatecznie wybrano k=5 po zastosowaniu metody łokcia i metryki Silhouette.

🚗 ANPR – Automatic Number Plate Recognition 🚗 ANPR – System Automatycznego Rozpoznawania Tablic

A high-performance License Plate Recognition system designed for automated access control. It combines a custom-trained YOLOv8 object detector with FastALPR for Optical Character Recognition (OCR), achieving an impressive 98% accuracy on Polish license plates.

Wysokowydajny system rozpoznawania tablic rejestracyjnych zaprojektowany do automatycznej kontroli dostępu. Łączy autorski model detekcji YOLOv8 z technologią FastALPR do rozpoznawania znaków (OCR), osiągając imponującą skuteczność 98% na polskich tablicach rejestracyjnych.

System Pipeline & Workflow Przepływ Pracy i Logika Systemu
1. Object Detection (YOLOv8) 1. Detekcja Obiektów (YOLOv8)

Locating the license plate in the frame. Lokalizacja tablicy rejestracyjnej na klatce.

  • Custom Weights: Utilizes a YOLOv8 model (`best.pt`) trained specifically on Polish license plates.
  • ROI Extraction: Crops the Region of Interest with adaptive padding to ensure full character visibility.
  • Autorskie Wagi: Wykorzystuje model YOLOv8 (`best.pt`) wytrenowany specjalnie na polskich tablicach.
  • Ekstrakcja ROI: Wycinanie obszaru zainteresowania z adaptacyjnym marginesem, aby zapewnić pełną widoczność znaków.
2. High-Precision OCR 2. Wysokoprecyzyjny OCR

Reading text from the cropped image. Odczytywanie tekstu z wyciętego obrazu.

  • FastALPR: Implements an advanced OCR solution based on LPRNet/ViT.
  • Performance: Significantly outperforms standard text-reading solutions like Tesseract or EasyOCR.
  • FastALPR: Implementacja zaawansowanego rozwiązania OCR opartego na architekturze LPRNet/ViT.
  • Wydajność: Znacząco przewyższa standardowe narzędzia do czytania tekstu, takie jak Tesseract czy EasyOCR.
3. Smart Access Logic 3. Logika Kontroli Dostępu

Simulating a parking barrier. Symulacja szlabanu parkingowego.

  • Text Cleaning: Filters output to purely alphanumeric characters.
  • Database Verification: Cross-references detected plates against an "Authorized Vehicles" whitelist.
  • Gate Simulation: Determines whether to open the virtual barrier based on authorization.
  • Czyszczenie Tekstu: Filtrowanie wyjścia do czysto alfanumerycznych znaków.
  • Weryfikacja Bazy: Porównywanie wykrytych tablic z "białą listą" uprawnionych pojazdów.
  • Symulacja Bramy: Podejmowanie decyzji o otwarciu wirtualnego szlabanu na podstawie autoryzacji.
4. Video & UI Processing 4. Przetwarzanie Wideo i UI

Real-time visual feedback. Informacje zwrotne w czasie rzeczywistym.

  • Stream Support: Configurable input for webcam (`0`) or pre-recorded video (`.mp4`).
  • Dynamic UI: Draws bounding boxes, reads, and visual status indicators (Green/Red) directly on the frame.
  • Optimization: Frame skipping logic ensures real-time processing capabilities.
  • Obsługa Strumieni: Konfigurowalne wejście dla kamery (`0`) lub nagrania wideo (`.mp4`).
  • Dynamiczne UI: Rysowanie ramek, odczytów i statusów wizualnych (Zielony/Czerwony) bezpośrednio na klatce.
  • Optymalizacja: Logika pomijania klatek gwarantująca płynne przetwarzanie obrazu.
